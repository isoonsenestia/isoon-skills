# i18n — next-intl

All repos use `next-intl` **except NESTA_UI**.

## Pages Router

`getStaticProps` + `useTranslations()`:

```tsx
// pages/appointments.tsx
export async function getStaticProps({ locale }: { locale: string }) {
  return {
    props: {
      messages: (await import(`../../locales/${locale}.json`)).default,
    },
  }
}

const AppointmentsPage: NextPageWithLayout = () => {
  const t = useTranslations()
  return <h1>{t('appointments_page.title')}</h1>
}

AppointmentsPage.getLayout = (page) => <Layout>{page}</Layout>
export default AppointmentsPage
```

## App Router

Layout provides `NextIntlClientProvider`; components call `useTranslations()`. Messages load once in `app/[locale]/layout.tsx` (or root layout). No `getStaticProps`.

```tsx
"use client"
import { useTranslations } from "next-intl"

export default function AppointmentCard() {
  const t = useTranslations("appointment_card")
  return <span>{t("title")}</span>
}
```

## Rule

Every user-visible string goes through `t()`. Never hardcode Thai or English strings in JSX. Add new keys to `locales/th.json` and `locales/en.json` (if present in this repo).
